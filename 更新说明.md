# 更新说明 - V2.0

## 重要更新

### V2.0 (2026-03-11) - 新增WiFi控制模式
**新增功能**：
- 支持WiFi远程控制模式
- 支持ESP8266-01S模块（简单控制）
- 支持ESP32-S3-CAM模块（带实时视频流）
- 网页控制界面，支持键盘操作（W/A/S/D/X/Q）
- ESP32-S3-CAM提供240x240实时视频流
- 单次点击模式，200ms自动停止保护

**模式切换优化**：
- SELECT键：在遥控模式和巡线模式之间切换
- START键：在遥控/巡线模式与WiFi模式之间切换
- LED指示：遥控模式熄灭，巡线模式常亮，WiFi模式每秒闪烁

**新增文件**：
- `ESP8266_WiFi.cpp/h`: WiFi模块驱动
- `Rzhl_connect/esp8266_webserver/`: ESP8266程序和说明
- `Rzhl_connect/esp32s3_cam/`: ESP32-S3-CAM程序和说明
- `测试指南.md`: 完整测试流程和故障排查

**技术特性**：
- ESP32-S3-CAM使用RGB565格式，实时转换为JPEG
- 硬件串口通信（115200波特率）
- 命令确认机制（ACK/No ACK）
- 自动停止保护（200ms超时）

**重要配置**：
- ESP32-S3-CAM必须使用正确的Arduino IDE分区方案
- Partition Scheme: "Huge APP (3MB No OTA/1MB SPIFFS)"
- PSRAM: "OPI PSRAM"
- 详见：`Rzhl_connect/esp32s3_cam/ESP32-S3-CAM使用说明.md`

### V1.4 (2026-03-09) - 修复按键检测bug
**问题描述**：
- START和SELECT键按下后没有任何反应
- 用户怀疑是按键映射问题

**问题分析**：
- 在ps2.cpp的`ps2_key_serch()`函数中发现bug
- 按键检测循环从`index=4`开始，到`index<16`结束
- 导致MASK数组的前4个元素（索引0-3）未被检测
- 对应的按键：SELECT(索引0)、L3(索引1)、R3(索引2)、START(索引3)

**修复方法**：
```cpp
// 修改前：
for(index=4; index<16; index++)

// 修改后：
for(index=0; index<16; index++)
```

**测试结果**：
- START和SELECT键现在都能正常工作
- 模式切换功能完全正常
- 灯光提示正常

## 使用方法

### 测试WiFi控制功能（新增）

#### 选项A：使用ESP8266-01S（无视频）
1. **硬件连接**
   - ESP8266 TX → Arduino Pin 3
   - ESP8266 RX → Arduino Pin 2（需要电压分压到3.3V）
   - ESP8266 VCC → 3.3V
   - ESP8266 GND → Arduino GND
   - ESP8266 CH_PD → 3.3V

2. **软件配置**
   - 打开`Rzhl_connect/esp8266_webserver/esp8266_webserver.ino`
   - 修改WiFi配置（第10-11行）
   - 上传到ESP8266模块

3. **测试**
   - 断开USB线，使用电池供电
   - 按START键进入WiFi模式（灯光闪3次）
   - 在浏览器输入ESP8266的IP地址
   - 使用网页按钮或键盘控制

#### 选项B：使用ESP32-S3-CAM（带视频流，推荐）
1. **硬件连接**
   - ESP32-S3 TX (GPIO1) → Arduino Pin 0 (RX)
   - ESP32-S3 RX (GPIO2) → Arduino Pin 1 (TX)
   - ESP32-S3 VCC → 5V独立电源（至少500mA）
   - ESP32-S3 GND → Arduino GND

2. **Arduino IDE配置（关键！）**
   ```
   板型：ESP32S3 Dev Module
   Partition Scheme: Huge APP (3MB No OTA/1MB SPIFFS)  ← 必须！
   PSRAM: OPI PSRAM  ← 必须！
   USB CDC On Boot: Disabled
   CPU Frequency: 240MHz (WiFi)
   ```

3. **软件配置**
   - 打开`Rzhl_connect/esp32s3_cam/esp32s3_cam.ino`
   - 修改WiFi配置（第12-13行）
   - 按上述配置设置Arduino IDE
   - 上传到ESP32-S3-CAM模块
   - 打开串口监视器（115200波特率）查看IP地址

4. **测试**
   - 断开USB线，使用电池供电
   - 按START键进入WiFi模式（灯光闪3次）
   - 在浏览器输入ESP32-S3的IP地址
   - 可看到实时视频流（240x240）
   - 使用网页按钮或键盘控制

5. **退出WiFi模式**
   - 网页按钮：点击"退出WiFi模式(Q)"
   - 键盘：按Q键
   - 手柄：按START键

详细说明请查看：
- ESP8266: `Rzhl_connect/esp8266_webserver/WiFi控制说明.md`
- ESP32-S3: `Rzhl_connect/esp32s3_cam/ESP32-S3-CAM使用说明.md`
- 完整测试流程：`测试指南.md`

### 串口调试（可选）

连接串口监视器（9600波特率）后，可以使用以下命令：
- `0`：进入遥控模式
- `1`：进入巡线模式
- `w`：进入WiFi模式（需要WiFi模块）
- `q`：退出WiFi模式
- `s`：显示传感器状态
- `m`：显示当前模式
- `d`：开启/关闭调试模式（显示所有按键）

## 传感器状态对照表

| 传感器状态 | 含义 | 小车动作 |
|-----------|------|---------|
| 1-0-0-1 | 中间两个检测到黑线 | 直行 |
| 1-1-0-1 | SENSOR2检测到黑线 | 右转（小弯） |
| 1-1-1-0 | SENSOR3检测到黑线 | 右转（大弯） |
| 1-0-1-1 | SENSOR1检测到黑线 | 左转（小弯） |
| 0-1-1-1 | SENSOR0检测到黑线 | 左转（大弯） |
| 1-1-1-1 | 全部未检测到 | 继续上次转向 |

**注意**：传感器值 0=检测到黑线（亮灯），1=白色地面（不亮）

## 故障排查

### 问题1：按START键没有灯光闪烁
**已在V1.4版本修复！**
- 原因：ps2.cpp中按键检测循环从索引4开始
- 修复：改为从索引0开始
- 现在START和SELECT键都能正常工作

### 问题2：灯光闪烁但小车不动
- 打开串口监视器查看传感器状态
- 检查传感器是否正确连接到引脚2-5
- 将小车放在黑线上，观察传感器状态是否变化
- 如果传感器状态一直是1-1-1-1，说明传感器未检测到黑线

### 问题3：小车运动方向错误
- 这是已知问题，Vehicle_Chassis函数命名与实际动作可能相反
- 需要修改Vehicle_Chassis.cpp中的函数实现
- 或者检查电机接线是否正确

### 问题4：串口无输出
- 检查波特率是否设置为9600
- 检查USB连接是否正常
- 重新上传代码

### 问题5：WiFi模式无法连接（新增）
**ESP8266常见问题**：
- 是否已断开USB线？（必须断开）
- ESP8266是否正确连接到Pin 2和Pin 3？
- ESP8266是否使用3.3V供电？（不能用5V）
- WiFi路由器是否是2.4GHz？（不支持5GHz）

**ESP32-S3-CAM常见问题**：
- Arduino IDE分区方案是否设置为"Huge APP"？
- PSRAM是否设置为"OPI PSRAM"？
- USB CDC On Boot是否设置为"Disabled"？
- ESP32-S3是否使用独立5V供电（至少500mA）？
- 串口监视器是否显示"Camera initialized"？

**调试方法**：
- ESP8266：使用USB转TTL模块查看串口输出
- ESP32-S3：直接查看串口监视器（115200波特率）

### 问题6：WiFi命令发送但小车不动（新增）
- 页面是否显示"Sent: x | OK"？
  - 如果显示"No ACK"，说明Arduino未收到命令
  - 检查TX/RX连接是否正确（TX→RX，RX→TX）
- 是否已进入WiFi模式？
  - LED应该每秒闪烁
- 是否按了START键进入WiFi模式？
  - 必须先进入WiFi模式才能控制

### 问题7：ESP32-S3-CAM编译错误"frame buffer malloc failed"（新增）
**原因**：Arduino IDE分区方案设置错误

**解决方法**：
1. 工具 → Partition Scheme → 选择"Huge APP (3MB No OTA/1MB SPIFFS)"
2. 工具 → PSRAM → 选择"OPI PSRAM"
3. 重新编译上传

详细故障排查请查看`测试指南.md`
