# ESP8266 WiFi控制模块使用说明

## 硬件连接

### ESP8266-01S模块连接到Arduino UNO
```
Arduino UNO    ESP8266-01S
3.3V       →   VCC (3V3)
GND        →   GND
Pin 0 (RX) →   TX
Pin 1 (TX) →   RX
不连接     ←   RST
不连接     ←   EN (内部上拉)
不连接     ←   IO0
不连接     ←   IO2
```

**重要提示：**
1. ESP8266工作电压为3.3V，VCC必须连接到Arduino的3.3V引脚
2. 使用硬件串口（Pin 0/1）时，上传代码前必须断开ESP8266的TX线（从Arduino Pin 0断开）
3. 上传完成后再重新连接TX线
4. 使用硬件串口后，无法同时使用USB串口监视器进行调试

## 软件配置

### 1. 修改WiFi配置
打开 `ESP8266_WiFi.cpp` 文件，修改以下内容：

```cpp
const char* WIFI_SSID = "YourWiFiSSID";        // 修改为你的WiFi名称
const char* WIFI_PASSWORD = "YourPassword";     // 修改为你的WiFi密码
```

### 2. 上传代码
**重要：上传代码时必须断开ESP8266的TX线！**

步骤：
1. 断开ESP8266 TX与Arduino Pin 0的连接
2. 使用Arduino IDE打开 `Rzhl_car.ino`
3. 选择正确的板型和端口
4. 点击上传
5. 上传完成后，重新连接ESP8266 TX到Arduino Pin 0
6. 按Arduino的复位按钮重启系统

### 3. 系统启动
- 系统启动时，LED会闪烁3次表示初始化完成
- 默认进入遥控模式

## 使用方法

### 通过PS2遥控器切换到WiFi模式
1. 启动小车，默认处于遥控模式（LED每秒闪烁1次）
2. 按下PS2遥控器的 **START** 键
3. 小车灯光闪烁3次，表示进入WiFi模式并开始初始化ESP8266
4. 等待约10-15秒，ESP8266连接到WiFi并启动服务器
5. 再次按 **START** 键可返回遥控模式（灯光闪烁2次）

**注意：** 由于使用硬件串口，无法通过串口监视器查看状态信息。请通过LED闪烁次数判断模式：
- 遥控模式：LED每秒闪烁1次
- 巡线模式：LED常亮
- WiFi模式：进入时闪烁3次

### 获取ESP8266的IP地址

由于串口被占用，有以下几种方法获取IP地址：

**方法1：使用路由器管理界面**
1. 登录你的WiFi路由器管理页面
2. 查看已连接设备列表
3. 找到名为"ESP_XXXXXX"的设备，记录其IP地址

**方法2：使用手机APP**
1. 下载"Fing"或"Net Analyzer"等网络扫描APP
2. 扫描局域网设备
3. 找到ESP8266的IP地址（通常显示为Espressif设备）

**方法3：尝试常见IP**
- 192.168.1.100 ~ 192.168.1.200
- 192.168.0.100 ~ 192.168.0.200
- 根据你的路由器网段尝试

## 网页控制

### 1. 打开控制页面
在同一WiFi网络下的电脑或手机浏览器中输入ESP8266的IP地址，例如：
```
http://192.168.1.100
```

### 2. 控制小车
控制页面提供两种方式：

**方式1：点击按钮**
- ↑ 按钮：前进
- ↓ 按钮：后退
- ← 按钮：左转
- → 按钮：右转
- ■ 按钮：停止

**方式2：键盘控制**
- W 或 ↑ 键：前进
- S 或 ↓ 键：后退
- A 或 ← 键：左转
- D 或 → 键：右转
- 空格键：停止

## 模式说明

### 三种工作模式
1. **遥控模式 (MODE_REMOTE)**
   - 默认模式
   - 使用PS2遥控器控制
   - LED每秒闪烁一次

2. **巡线模式 (MODE_LINE_TRACK)**
   - 按SELECT键进入
   - 自动循迹
   - LED常亮

3. **WiFi模式 (MODE_WIFI)**
   - 按START键进入
   - 通过网页控制
   - 进入时LED闪烁3次

### 模式切换（仅通过PS2手柄）
- **START键**：在遥控模式/巡线模式 ↔ WiFi模式之间切换
- **SELECT键**：在遥控模式 ↔ 巡线模式之间切换（WiFi模式下无效）

**注意：** 由于使用硬件串口连接ESP8266，串口命令功能已禁用，所有操作必须通过PS2手柄完成。

## 串口调试命令

**注意：由于使用硬件串口（Pin 0/1）连接ESP8266，串口调试功能已禁用。**

如果需要调试，可以：
1. 使用SoftwareSerial连接ESP8266（需修改代码）
2. 使用Arduino Mega等多串口板
3. 通过LED闪烁次数判断系统状态

## 故障排除

### 上传代码失败
**原因：** ESP8266的TX线连接到Arduino Pin 0，干扰了串口通信
**解决：** 上传代码前必须断开ESP8266 TX与Arduino Pin 0的连接

### ESP8266无响应
1. 检查供电是否为3.3V（不能是5V）
2. 检查TX/RX连接是否正确（TX→Pin0, RX→Pin1）
3. 上传代码后是否重新连接了TX线
4. 按Arduino复位按钮重启系统

### 无法连接WiFi
1. 确认WiFi名称和密码正确（区分大小写）
2. 确认WiFi为2.4GHz频段（ESP8266不支持5GHz）
3. 等待足够的初始化时间（约10-15秒）
4. 检查路由器是否限制了设备连接

### 不知道ESP8266的IP地址
1. 使用路由器管理界面查看已连接设备
2. 使用手机APP（如Fing）扫描网络
3. 尝试常见IP段：192.168.1.100-200 或 192.168.0.100-200

### 网页无法打开
1. 确认手机/电脑连接到同一WiFi
2. 确认IP地址正确
3. 尝试ping ESP8266的IP地址
4. 检查路由器是否开启了AP隔离
5. 等待ESP8266完全初始化（进入WiFi模式后等待15秒）

### 控制延迟或无响应
1. 检查WiFi信号强度
2. 减少与路由器的距离
3. 重启ESP8266模块（按Arduino复位按钮）
4. 检查是否有多个设备同时连接
2. 确认WiFi为2.4GHz频段（ESP8266不支持5GHz）
3. 检查串口监视器的错误信息

### 网页无法打开
1. 确认手机/电脑连接到同一WiFi
2. 确认IP地址正确
3. 尝试ping ESP8266的IP地址
4. 检查路由器是否开启了AP隔离

### 控制延迟或无响应
1. 检查WiFi信号强度
2. 减少与路由器的距离
3. 重启ESP8266模块

## 技术参数

- WiFi协议：802.11 b/g/n
- 工作频率：2.4GHz
- 服务器端口：80
- 控制速度：600 (可在代码中修改)
- 通信波特率：115200 (ESP8266) / 9600 (Arduino串口监视器)

## 安全提示

1. ESP8266模块工作电压为3.3V，请勿直接连接5V电源
2. 建议在RX引脚使用分压电路保护ESP8266
3. WiFi密码会明文存储在代码中，请注意安全
4. 建议使用独立的WiFi网络或访客网络
