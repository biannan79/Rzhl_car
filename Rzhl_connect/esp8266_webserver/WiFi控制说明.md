# Rzhl智能小车 WiFi控制使用说明

## 硬件连接

### ESP8266-01S 接线
```
ESP8266 VCC  → Arduino 3.3V (重要：不能接5V！)
ESP8266 GND  → Arduino GND
ESP8266 TX   → Arduino Pin 0 (RX)
ESP8266 RX   → Arduino Pin 1 (TX)
ESP8266 CH_PD → Arduino 3.3V
```

**注意事项：**
- ESP8266工作电压是3.3V，接5V会烧坏模块
- 上传Arduino代码时必须断开ESP8266的TX和RX连接线
- ESP8266只支持2.4GHz WiFi，不支持5GHz

## 软件配置

### 1. 配置WiFi信息

编辑 `esp8266_webserver.ino` 文件的第10-11行：

```cpp
const char* ssid = "你的WiFi名称";
const char* password = "你的WiFi密码";
```

### 2. 上传代码

**步骤：**
1. 断开ESP8266的TX/RX连接线
2. 打开Arduino IDE，选择板子 "Generic ESP8266 Module"
3. 上传 `esp8266_webserver.ino` 到ESP8266
4. 打开Arduino IDE，选择板子 "Arduino UNO"
5. 上传 `Rzhl_car.ino` 到Arduino UNO
6. 重新连接ESP8266的TX/RX线

## 使用方法

### 进入WiFi模式

1. 给小车上电（默认进入遥控模式，LED灯熄灭）
2. 按下PS2手柄的 **START** 键
3. 小车灯光闪烁3次，然后LED开始1秒闪烁一次（表示进入WiFi模式）
4. 等待约10秒，ESP8266连接到WiFi

### 查找小车IP地址

**方法1：查看路由器**
- 登录你的WiFi路由器管理页面
- 查看已连接设备列表
- 找到ESP8266的IP地址（通常是 192.168.x.x）

**方法2：使用串口监视器（需要单独测试ESP8266）**
- 断开ESP8266与Arduino的连接
- 用USB转TTL模块连接ESP8266
- 打开串口监视器（115200波特率）
- 查看输出的IP地址

### 控制小车

1. 在手机或电脑浏览器中输入ESP8266的IP地址
   - 例如：`http://192.168.43.40`

2. 打开控制页面后，可以使用：
   - **网页按钮**：点击方向按钮控制
   - **键盘控制**：
     - W 或 ↑ ：前进
     - S 或 ↓ ：后退
     - A 或 ← ：左转
     - D 或 → ：右转
     - X 或 空格：停止
     - Q：退出WiFi模式

3. 每次点击或按键，小车会移动约200毫秒后自动停止

### 退出WiFi模式

**方法1：网页按钮**
- 点击页面上的"退出WiFi模式"按钮

**方法2：键盘**
- 按 Q 键

**方法3：PS2手柄**
- 按 START 键

退出后，小车灯光闪烁2次，LED熄灭，返回遥控模式。

## 控制特性

### 单次点击模式
- 每次点击按钮或按键，小车只移动约200ms
- 如需持续移动，需要连续点击
- 200ms内无新命令，小车自动停止

### 自动停止保护
- 如果WiFi连接断开或网页关闭
- 小车会在200ms后自动停止
- 防止失控

## 状态指示

### LED灯状态
- **熄灭**：遥控模式
- **1秒闪烁**：WiFi模式
- **常亮**：巡线模式

### 尾灯状态
- **熄灭**：停止
- **两个都亮**：后退
- **左侧闪烁**：左转
- **右侧闪烁**：右转

## 常见问题

### Q1: 网页打不开
- 检查手机/电脑是否连接到同一个WiFi
- 确认IP地址是否正确
- 等待ESP8266完全连接到WiFi（约10秒）
- 检查路由器是否开启了AP隔离

### Q2: 小车不动
- 确认已按START键进入WiFi模式（LED闪烁）
- 检查网页是否显示 "Sent: w | OK"
- 如果显示 "No ACK"，说明Arduino没有响应，检查连线

### Q3: 无法上传代码
- 上传前必须断开ESP8266的TX/RX连接线
- 确认选择了正确的板子类型
- 检查USB连接是否正常

### Q4: ESP8266连接不上WiFi
- 确认WiFi是2.4GHz频段（不支持5GHz）
- 检查WiFi名称和密码是否正确
- 确认ESP8266供电正常（3.3V）

### Q5: 小车移动时间太短/太长
- 修改 `Rzhl_car.ino` 第31行的 `WIFI_TIMEOUT_TICKS` 值
- 默认值10 = 200ms
- 增大数值可延长移动时间（例如15 = 300ms）

## 技术参数

- WiFi频段：2.4GHz
- 通信波特率：115200
- 控制延迟：< 100ms
- 自动停止时间：200ms
- 电机速度：600 (范围0-1000)
- 网页端口：80

## 安全提示

1. 首次使用建议在空旷场地测试
2. 确保WiFi信号稳定，避免失控
3. 注意小车移动范围，避免碰撞
4. 不要在WiFi模式下长时间不操作（会自动停止）
5. 如遇异常，立即按X键或断电
