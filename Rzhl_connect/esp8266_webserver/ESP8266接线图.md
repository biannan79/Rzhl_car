## 引脚对应关系

| Arduino UNO | ESP8266-01S | 说明 |
|-------------|-------------|------|
| 3.3V        | VCC (3V3)   | 电源正极（必须3.3V） |
| GND         | GND         | 电源地 |
| Pin 0 (RX)  | TX          | 串口接收 ← ESP发送 |
| Pin 1 (TX)  | RX          | 串口发送 → ESP接收 |
| 不连接      | RST         | 复位（内部上拉） |
| 不连接      | EN (CH_PD)  | 使能（内部上拉） |
| 不连接      | IO0         | GPIO0（内部上拉） |
| 不连接      | IO2         | GPIO2（内部上拉） |

## 重要提示

### ⚠️ 电压警告
- **ESP8266工作电压为3.3V**
- **绝对不能连接5V，否则会烧毁模块！**
- Arduino UNO的3.3V引脚电流较小（约50mA），如果ESP8266工作不稳定，建议使用外部3.3V稳压模块

### ⚠️ 上传代码注意事项
**上传Arduino代码时必须断开ESP8266的TX线！**

操作步骤：
1. 拔掉ESP8266 TX与Arduino Pin 0之间的连接线
2. 上传代码到Arduino
3. 上传完成后，重新连接ESP8266 TX到Arduino Pin 0
4. 按Arduino的复位按钮重启

原因：Arduino使用Pin 0/1作为USB串口通信，上传代码时ESP8266会干扰通信。

### 📌 引脚说明
- **Pin 0 (RX)**：Arduino的串口接收引脚，连接到ESP8266的TX
- **Pin 1 (TX)**：Arduino的串口发送引脚，连接到ESP8266的RX
- **EN (CH_PD)**：ESP8266的使能引脚，ESP-01S模块内部已上拉，无需外接
- **RST**：复位引脚，ESP-01S模块内部已上拉，无需外接

## 实物接线照片参考

建议使用杜邦线连接，接线时注意：
1. 先断开Arduino电源
2. 按照上表连接所有线
3. 仔细检查VCC是否连接到3.3V（不是5V）
4. 检查TX/RX是否交叉连接（Arduino RX ← ESP TX，Arduino TX → ESP RX）
5. 确认GND已连接
6. 上传代码前断开ESP8266 TX线
7. 上传完成后重新连接TX线
8. 给Arduino上电

## 供电方案

### 方案1：使用Arduino板载3.3V（简单但可能不稳定）
- 直接连接Arduino的3.3V引脚
- 适合测试，但ESP8266功耗较大时可能导致电压不足

### 方案2：使用外部3.3V稳压模块（推荐）
- 使用AMS1117-3.3或类似稳压模块
- 输入：Arduino 5V或外部电源
- 输出：稳定的3.3V给ESP8266
- 可提供更大电流，工作更稳定

### 方案3：使用独立3.3V电源
- 使用两节AA电池（2×1.5V=3V）
- 或使用锂电池+降压模块
- 注意：必须共地（GND连接）
