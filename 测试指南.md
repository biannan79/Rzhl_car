# Rzhl智能小车测试指南 V2.0

## 版本信息
- 版本：V2.0
- 更新日期：2026-03-11
- 主要功能：遥控模式、巡线模式、WiFi控制模式

## 重要说明
本系统支持三种控制模式，可通过PS2手柄按键或串口命令切换：
1. **遥控模式**（默认）：使用PS2无线手柄控制
2. **巡线模式**：使用红外传感器自动循线
3. **WiFi模式**：通过网页或键盘远程控制

## LED指示说明

### 板载LED（引脚13）状态
| LED状态 | 含义 | 持续时间 |
|---------|------|---------|
| 快闪3次 | 系统启动完成 | 启动时 |
| 熄灭 | 遥控模式 | 持续 |
| 常亮不闪 | 巡线模式 | 持续 |
| 每秒闪1次 | WiFi模式 | 持续 |
| 闪1次 | 进入巡线模式 | 瞬间 |
| 闪2次 | 退出巡线/WiFi模式 | 瞬间 |
| 闪3次 | 进入WiFi模式 | 瞬间 |

### 尾灯状态
| 尾灯状态 | 含义 |
|---------|------|
| 全灭 | 停止或前进 |
| 全亮 | 后退 |
| 左灯闪烁 | 左转 |
| 右灯闪烁 | 右转 |

## 完整测试流程

### 步骤1：上传代码
1. 用Arduino IDE打开Rzhl_car.ino
2. 连接USB线
3. 选择正确的板型和端口
4. 点击上传
5. 等待上传完成

### 步骤2：断开USB，使用电池
1. **拔掉USB线**（重要：使用WiFi模式时必须断开USB）
2. 给小车接上电池
3. 打开电源开关

### 步骤3：观察启动
1. 观察板载LED（引脚13）
2. 应该看到：**全身灯光闪烁2次**（系统就绪）
3. 然后LED熄灭（进入遥控模式）

### 步骤4：连接遥控器
1. 打开PS2遥控器
2. 等待遥控器指示灯正常（表示已连接）
3. 测试遥控功能，确认小车能正常移动

4. 观察尾灯是否根据动作正确显示

### 测试2：巡线模式
**进入方式**：
1. 在遥控模式下按**SELECT键**
2. 观察现象：
   - ✅ 全身灯光闪烁1次
   - ✅ 板载LED变为**常亮**

**功能测试**：
1. 将小车放在黑色轨道线上
2. 观察小车是否开始沿线行驶
3. 观察尾灯是否根据转向闪烁：

**退出方式**：

2. 观察现象：

   - ✅ 小车立即停止
   - ✅ 板载LED熄灭（返回遥控模式）

### 测试3：WiFi控制模式

**支持两种WiFi模块：**
- **ESP8266-01S**：简单控制（无视频）
- **ESP32-S3-CAM**：视频流+控制（推荐）

#### 选项A：使用ESP8266-01S（无视频）

**硬件准备：**
- ESP8266 TX → Arduino Pin 0 (RX)
- ESP8266 RX → Arduino Pin 1 (TX)
- ESP8266 GND → Arduino GND
- ESP8266 VCC → Arduino 3.3V（**不能接5V！**）
- ESP8266 CH_PD → 3.3V

**重要提示**：
- 必须断开USB线才能使用WiFi模式（硬件串口冲突）
- ESP8266必须使用3.3V供电
- 确保WiFi路由器是2.4GHz频段

**软件准备：**
1. 打开esp8266_webserver.ino
2. 修改WiFi配置（第10-11行）：
```cpp
const char* ssid = "你的WiFi名称";
const char* password = "你的WiFi密码";
```
3. 上传到ESP8266-01S模块

**详细说明：** 参见 `Rzhl_connect/esp8266_webserver/WiFi控制说明.md`

#### 选项B：使用ESP32-S3-CAM（带视频流）

**硬件准备：**
- ESP32-S3 VCC → 5V独立电源（至少500mA）
- ESP32-S3 GND → Arduino GND
- ESP32-S3 TX (GPIO1) → Arduino Pin 0 (RX)
- ESP32-S3 RX (GPIO2) → Arduino Pin 1 (TX)

**重要提示**：
- ESP32-S3需要独立5V供电，不要与Arduino共用电源
- 必须断开USB线才能使用WiFi模式
- 确保WiFi路由器是2.4GHz频段

**Arduino IDE配置（关键！）：**
```
板型：ESP32S3 Dev Module
USB CDC On Boot: Disabled
CPU Frequency: 240MHz (WiFi)
Partition Scheme: Huge APP (3MB No OTA/1MB SPIFFS)  ← 必须！
PSRAM: OPI PSRAM  ← 必须！
Upload Speed: 921600
```

**软件准备：**
1. 打开esp32s3_cam.ino
2. 修改WiFi配置（第10-11行）
3. 按上述配置设置Arduino IDE
4. 上传到ESP32-S3-CAM模块
5. 打开串口监视器（115200波特率）查看IP地址

**详细说明：** 参见 `Rzhl_connect/esp32s3_cam/ESP32-S3-CAM使用说明.md`

#### 通用测试步骤（ESP8266和ESP32-S3通用）

**进入WiFi模式：**
1. 确保已断开USB线，使用电池供电
2. 在遥控模式或巡线模式下按**START键**
3. 观察现象：
   - ✅ 全身灯光闪烁3次
   - ✅ 板载LED开始每秒闪烁
4. 等待5-10秒让WiFi模块连接WiFi

**查看IP地址：**
- ESP8266：查看路由器设备列表
- ESP32-S3：串口监视器显示IP地址（推荐）

**网页控制测试：**
1. 在手机或电脑浏览器输入WiFi模块的IP地址
2. ESP8266：看到控制页面
3. ESP32-S3：看到控制页面+实时视频流（240x240）
4. 测试按钮控制：
   - 点击↑(W)：小车前进约200ms后自动停止
   - 点击↓(S)：小车后退约200ms后自动停止
   - 点击←(A)：小车左转约200ms后自动停止
   - 点击→(D)：小车右转约200ms后自动停止
   - 点击■(X)：立即停止
5. 观察页面状态显示：
   - "Sent: w | OK"：命令发送成功，Arduino已确认
   - "Sent: w | No ACK"：命令发送但Arduino未响应

**键盘控制测试：**
1. 在控制页面上，直接按键盘按键：
   - W或↑：前进
   - S或↓：后退
   - A或←：左转
   - D或→：右转
   - X或空格：停止
   - Q：退出WiFi模式
2. 每次按键小车只动约200ms（单次点击模式）

**退出WiFi模式：**
- 网页按钮：点击"退出WiFi模式(Q)"
- 键盘：按Q键
- 手柄：按START键

**退出后现象：**
- ✅ 全身灯光闪烁2次
- ✅ 小车立即停止
- ✅ 板载LED熄灭（返回遥控模式）

## 故障排查

### 问题1：启动后LED不闪烁
**可能原因**：
- LED硬件损坏
- 代码未正确上传

**解决方法**：
1. 重新上传代码
2. 检查LED是否损坏
3. 即使LED不亮，也可以继续测试（用尾灯观察）

### 问题2：遥控器无法连接
**可能原因**：
- 遥控器电池没电
- PS2接收器接线错误
- 电源供电不足

**解决方法**：
1. 更换遥控器电池
2. 检查PS2接收器接线（DI=11, DO=9, CS=8, SCK=10）
3. 确认电池电量充足

### 问题3：按SELECT或START键没反应
**检查清单**：
1. 确认遥控器已正常连接
2. 确认其他按键能正常工作
3. 确认已上传最新版本代码

### 问题4：进入巡线模式但小车不动
**检查清单**：
1. ✅ LED是否常亮？（确认已进入巡线模式）
2. ✅ 小车是否放在黑线上？
3. ✅ 传感器LED是否亮起？（检测到黑线时应该亮）
4. ✅ 传感器接线是否正确？（引脚2-5）

**传感器测试**：
- 将小车放在黑线上
- 观察4个传感器的LED
- 中间两个传感器应该亮（检测到黑线）
- 两边传感器应该灭（检测到白色地面）

### 问题5：WiFi模式无法连接

**ESP8266常见问题：**
1. ✅ 是否已断开USB线？（必须断开）
2. ✅ ESP8266是否正确连接到Pin 0和Pin 1？
3. ✅ ESP8266是否使用3.3V供电？（不能用5V）
4. ✅ WiFi路由器是否是2.4GHz？（不支持5GHz）
5. ✅ WiFi名称和密码是否正确？

**ESP32-S3-CAM常见问题：**
1. ✅ Arduino IDE分区方案是否设置为"Huge APP"？
2. ✅ PSRAM是否设置为"OPI PSRAM"？
3. ✅ USB CDC On Boot是否设置为"Disabled"？
4. ✅ ESP32-S3是否使用独立5V供电（至少500mA）？
5. ✅ 串口监视器是否显示"Camera initialized"？

**调试方法：**
- ESP8266：使用USB转TTL模块查看串口输出
- ESP32-S3：直接查看串口监视器（115200波特率）

### 问题6：WiFi命令发送但小车不动
**检查清单**：
1. 页面是否显示"Sent: x | OK"？
   - 如果显示"No ACK"，说明Arduino未收到命令
   - 检查TX/RX连接是否正确（TX→RX，RX→TX）
2. 是否已进入WiFi模式？
   - 板载LED应该每秒闪烁
3. 是否按了START键进入WiFi模式？
   - 必须先进入WiFi模式才能控制

### 问题7：小车运动方向错误
**现象**：
- 应该前进却后退
- 应该左转却右转

**解决方法**：
需要修改Vehicle_Chassis.cpp或Rzhl_car.ino中的电机方向，告诉我具体的错误方向

## 串口调试命令（可选）

如果需要调试，可以在上传代码前启用串口调试：
1. 注释掉ESP8266_Init()
2. 取消注释Serial.begin(9600)
3. 上传代码后打开串口监视器（9600波特率）

**可用命令**：
- `0` - 切换到遥控模式
- `1` - 切换到巡线模式
- `m` - 显示当前模式
- `s` - 显示传感器状态
- `d` - 切换调试模式（显示所有按键）

**注意**：启用串口调试后WiFi功能将不可用（硬件串口冲突）
